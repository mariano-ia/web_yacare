Entregable 1

Nomenclatura

La base del Design System. Sin nombres consistentes, no hay librería posible.

01
El estado anterior

Nombres inconsistentes en Figma

Un componente mal nombrado no aparece en búsquedas de Figma y no puede usarse como librería. La nomenclatura es la base del DS.

Había mezcla de inglés y español, prefijos arbitrarios, paréntesis descriptivos, typos y frames sueltos sin convertir a sets.

settings/button→Button
Seachbar→Searchbar
button (destructive)→Button / type=Destructive
Zone Badge/→ZoneBadge
Input text→Input / type=Text
topbar (mobile)→TopbarMobile

Acciones por tipo

Solo renombrar

Unificar sets

Dos o más component sets que deberían ser uno solo con propiedad type.

Convertir a set

El componente existe como frame suelto. No está disponible como librería.

OK

Nombre y estructura correctos. Sin acción urgente. Se preserva tal como está.

Formato

Estructura replicable para el equipo

Cada documento sigue la misma estructura para que cualquier persona del equipo pueda continuar documentando nuevos componentes.

La documentación reduce errores de implementación, acelera el onboarding y es la base para la librería de código.

  • Descripción del componente y su propósito en el producto
  • Cuándo usarlo con casos de uso concretos y contexto del producto
  • Cuándo NO usarlo con la alternativa recomendada en cada caso
  • Ejemplo en contexto con captura de pantalla del producto real

03
El argumento

De los planos al catálogo de materiales

Reutilización directa

El equipo de dev importa componentes en vez de fabricarlos cada vez. Menos divergencias, menos tiempo.

Propagación automática

Un cambio en el DS (color, spacing, radio) se propaga a todo el producto de forma consistente.

Fuente de verdad

Diseño, dev y producto ven lo mismo. Sin interpretaciones ni versiones desactualizadas.

Onboarding sin Figma

PM, QA y nuevos miembros del equipo pueden revisar el portal sin conocimientos de Figma.

El prototipo

Lo que ya existe hoy

El portal es un prototipo funcional que muestra la dirección. No es un producto terminado.

  • Tokens de color reales extraídos directamente del DS de Figma (Accent, Surface, Content, Semantic)
  • Tipografía completa con todos los estilos del DS renderizados en Space Grotesk y DM Sans
  • Button interactivo con propiedades, variantes visuales, playground y guía de cuándo usar
  • Sidebar con dirección el resto de componentes marcados como "próximamente" para mostrar el roadmap
